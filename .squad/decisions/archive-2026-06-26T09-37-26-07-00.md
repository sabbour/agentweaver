#### 2026-06-17: Coordinator runs as a MAF orchestration agent, not a new runtime
**By:** Morpheus  
**Status:** Approved

Model the coordinator as a MAF agent hosted in its own observable run, with a new
`CoordinatorOrchestratorExecutor` inside a MAF `Workflow` that dispatches child runs through the
existing `RunWorkflowFactory`/`RunOrchestrator` pipeline.

**Rationale:** Principle I/XI require building on MAF and not reimplementing the agent loop or
governance. Keeping fan-out/serialize inside a MAF executor keeps telemetry, guardrails, and
auditability in the runtime layer rather than ad hoc threads.

#### 2026-06-17: Subagents are first-class child runs via a new `Run.ParentRunId`/`SubtaskId`
**By:** Morpheus  
**Status:** Approved

tagged with nullable `ParentRunId` (coordinator) and `SubtaskId`. Read-only observation is the
existing `/api/runs/{id}/stream` SSE endpoint.

**Rationale:** FR-015 requires own step stream, own RAI, own sandbox per subagent. Reusing the
existing run primitives (Features 001/002) with two nullable linkage fields avoids a parallel
run type and is fully backward compatible.

#### 2026-06-17: Phase 2 smoke-test defect B ‚Äî retrievable failed child run on pre-start failure

#### 2026-06-17: Child runs use a TRIMMED pipeline that terminates assemble-ready after RAI (B1)
**By:** Tank  
**Status:** Approved

Replaced the claim that children "reuse the existing pipeline unchanged" with a
designed child-run workflow variant ‚Äî `BuildChildWorkflow()` (or a `ParentRunId != null` branch
in `RunWorkflowFactory.BuildWorkflow`) that runs `agentInputStorer -> agent -> RAI` plus the
existing RAI revise loop, then routes RAI's non-revision, diff-present output to a new
`childAssembleReady` terminal executor instead of `reviewAdapter`. No `review-gate` RequestPort,
no `MergeExecutor`, no `ScribeTurnExecutor` on the child path. Added a `Subtask.Status =
assemble_ready` state. Pulled this into Phase 2 as a prerequisite of the first dispatch.

**Rationale:** Today `BuildWorkflow` hardwires `agent -> RAI -> review -> merge -> scribe` for every run
(`RunWorkflowFactory.cs:485-531`). Any child started via `RunOrchestrator.StartRunAsync` would
traverse the whole graph, pause at its OWN human review gate, and merge to the originating branch
‚Äî N children = N human gates + N merges, violating FR-021/SC-004 ("exactly once over collective
output"). Deferring the short-circuit to Phase 3 while calling Phase 2 "shippable" was
contradictory and would have required fake child termination in Phase 2 (Principle VII). The
trimmed pipeline must exist before the first real dispatch.

#### 2026-06-17: Steering is NEW infrastructure gated on a feasibility spike, not reuse (B2)
**By:** Tank  
**Status:** Approved

Reclassified redirect/amend/pause/stop steering as new infrastructure. Honest

mid-turn redirect/amend/pause hook, and `RunOrchestrator.StartRevisionAsync` only fires when a
run is already PAUSED at its review gate after a turn completes. Presenting new control surfaces
as "reuse" was inaccurate and unbuildable as written.

#### 2026-06-18: Phase 3 Collective Assembly ‚Äî service-driven, not a MAF graph
**By:** Morpheus (Backend Engineer)
**Branch:** 008-coordinator-agent
**Date:** 2026-06-18
**Status:** Implemented (backend complete; frontend surfacing deferred)

Phase 3 replaces the `awaiting_assembly` dead end with a collective pipeline over the combined output
of all children: integration branch ‚Üí collective RAI ‚Üí one human review ‚Üí one merge ‚Üí one scribe.

**Key decisions:**

- **D3 (Architecture):** `CoordinatorAssemblyService` drives the collective pipeline as a sequential
  service (not a MAF graph). The collective pipeline starts from already-assembled git state; reusing
  the single-run MAF graph would run an extra agent turn and route review feedback to the wrong actor.
  Heavy operations reused: `WorktreeManager.BuildIntegrationBranch`, `RaiTurnExecutor`,
  `ScribeTurnExecutor`, `WorktreeManager.MergeWorktree`, `RepositoryMergeLock`.

- **D1 (Combined artifact):** A git integration branch `scaffolder/integration/{coordinatorRunId}` is
  built from the coordinator's originating branch by merging eligible child branches in dependency
  (topological) order. Persisted in `WorkPlan.IntegrationBranch`.

- **D2 (No partial assembly):** Every subtask must be assembly-eligible before assembly starts. Any
  failed / rai_flagged / pending / blocked subtask, or a merge conflict, emits
  `coordinator.assembly_blocked` and stops.

- **D4 (Exactly-once):** `CoordinatorAssemblyStore.TryStartAssemblyAsync` uses a DB compare-and-swap
  (`ExecuteUpdateAsync` with `WHERE Status = 'awaiting_assembly'`) to atomically transition to
  `assembling`. An in-memory `_active` guard is the cheap first line.

- **D5/D6 (Review gate + rejection inference):** `POST /api/runs/{coordinatorRunId}/assembly/review`.
  On `request_changes`, the coordinator infers which children to redo from files referenced in
  reviewer feedback (explicit `target_files` ‚à™ path-like tokens parsed from free-text). Children whose
  touched-files intersect the inferred set, plus their transitive dependents, are re-dispatched.
  Fallback: re-dispatch all if no file can be inferred.

- **Node-flip:** `WorkPlan.AssemblyStage` (rai/review/merge/scribe/done) persisted per stage;
  `CoordinatorGraphDescriptor.Build` renders each planned assembly node as `kind="live"` once its
  ordinal ‚â§ current stage ordinal.

- **Status transitions:** `dispatching ‚Üí awaiting_assembly ‚Üí assembling ‚Üí in_review ‚Üí assembling ‚Üí complete`.
  Terminal: `assembly_blocked`, `assembly_failed`, `assembly_declined`.

- **Events (13):** `coordinator.assembly_started/rai_started/rai_completed/review_requested/
  review_approved/changes_requested/merge_started/merge_completed/merge_failed/scribe_started/
  scribe_completed/completed/blocked`. Registered in `packages/Scaffolder.Domain/EventTypes.cs`.

**Bug fixes found while testing:** `CoordinatorAssemblyStore` and subtask-reset used inline
`DateTimeOffset.UtcNow` inside `ExecuteUpdateAsync` ‚Äî EF cannot translate; fixed by capturing a
local `now` before the call.

**Rationale:** Service over MAF graph because the collective pipeline has no agent turn to anchor;
review routes back to the coordinator (not in-loop to an agent); exactly-once, integration-branch
build, and HITL resume are coordinator-owned concerns.
