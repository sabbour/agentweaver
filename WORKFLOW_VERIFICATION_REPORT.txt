    YAML → C# Property Mapping:
    snake_case (YAML) ↔ PascalCase (C#)
    - trigger_type → TriggerType
    - gate_kind → GateKind
    - workflow_id → WorkflowId
    - task_id → TaskId

5.2 YAML DTO CLASSES (lines 223–265)
    WorkflowYamlDto:
    - id, name, description (strings)
    - trigger: TriggerYamlDto
    - start: string (entry node id)
    - nodes: List<NodeYamlDto>
    - edges: List<EdgeYamlDto>

    TriggerYamlDto:
    - type: string (manual|heartbeat|event)
    - event: string (task-added-to-ready)

    NodeYamlDto:
    - id, type, label, role, kind, gate_kind
    - agent, prompt, target
    - steps: List<string> (for serial nodes)
    - branches: List<string> (for check nodes)

    EdgeYamlDto:
    - from, to (node ids)
    - when: string (verdict/predicate, e.g., "approved")

5.3 EXAMPLE YAML STRUCTURE
    id: my-workflow
    name: My Custom Workflow
    description: A declarative workflow
    trigger:
      type: heartbeat
    start: agent
    nodes:
      - id: agent
        type: prompt
        label: Agent
        role: agent
        prompt: Do the task
      - id: review
        type: check
        gate_kind: human-review
        branches:
          - approved
          - request-changes
    edges:
      - from: agent
        to: review
      - from: review
        to: done
        when: approved
